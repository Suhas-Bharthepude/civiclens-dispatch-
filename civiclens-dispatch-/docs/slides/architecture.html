The Problem
Emergency dispatch is
a manual bottleneck
Dispatch centers receive hundreds of unstructured reports per shift — text messages, voice calls, photos — from citizens, sensors, and field units.

Every report must be manually read, classified, prioritized, and summarized before a dispatcher can act. Under pressure, this is slow and error-prone.

CivicLens Dispatch automates that triage pipeline using multimodal AI — so dispatchers spend less time reading reports and more time making decisions.

Key Decisions
Why I chose
what I chose
Framework
FastAPI
Async-native — essential for non-blocking background AI tasks. Flask is sync by default. Django is overkill for an API-only backend. Automatic Swagger docs and Pydantic validation included.
Database
SQLite → PostgreSQL
SQLite for development: zero config, file-based, fast. The ORM layer (SQLAlchemy) means swapping to PostgreSQL for production is a config change, not a rewrite.
AI Strategy
Zero-Shot Classification
No labeled training data for emergency dispatch exists publicly. Zero-shot models (BART-MNLI) understand language well enough to classify incident types without task-specific fine-tuning.
Each decision traded short-term complexity for long-term maintainability — and each one I can defend with a concrete technical reason.
