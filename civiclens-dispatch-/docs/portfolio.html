What It Does

How it works

📥
Takes reports in any format
A citizen (or sensor, or dispatcher) submits a report. That can be a text description, a voice recording, or a photo — or all three. The system handles whichever ones are provided.
🤖
AI fills in the details automatically
Four AI models run in the background after a report is submitted. They transcribe audio, describe images, label the incident type (fire, medical, traffic, etc.), score how urgent it is, and write a one-sentence summary. This takes about 5 seconds.
📊
Dispatchers see a sorted, labeled feed
The dashboard shows all incidents ranked by urgency score — highest risk at the top. Dispatchers can filter by incident type or severity, search by keyword, and click any row to see the full AI analysis.

Under the Hood

How the AI pipeline works

Phase 1 — runs first
🎤 Audio transcription
If audio was uploaded, Whisper converts it to text. This runs at the same time as image analysis below.
📷 Image analysis
If a photo was uploaded, DETR describes what's in it (vehicles, smoke, people, etc.).
Phase 2 — runs after Phase 1
🏷️ Incident classification
Labels the incident as fire, medical, traffic, crime, noise, infrastructure, or other — plus a severity level (low / medium / high).
📝 Summary generation
Writes a one or two sentence summary of what happened, pulling from the description, transcript, and image caption.
🎯 Urgency score
Assigns a number from 0 to 1 representing how urgent the incident is. This is what the dashboard sorts by.
Phase 1 tasks run at the same time as each other. Phase 2 tasks also run at the same time as each other. Phase 2 waits for Phase 1 to finish first because it needs the transcript. Total time: about 5 seconds.
Built With

Tech stack

FastAPI
The backend server. Handles all API requests, validates incoming data, and kicks off the AI pipeline in the background after each new incident.
React
The dispatcher dashboard. Fetches incidents from the backend and displays them in a live, filterable table with a detail panel on the side.
asyncio
Python's built-in tool for running tasks at the same time. Used so the AI models in each phase don't wait for each other unnecessarily.
Hugging Face
Hosts the four AI models used in the pipeline. Requests are sent over their API — no GPU or local model setup required.
SQLite + SQLAlchemy
The database where incidents and their AI-generated fields are stored. Indexed on type, severity, risk score, and date for fast filtering.
pytest
The test suite — 20 integration tests covering the full pipeline, all error conditions, and query performance.
